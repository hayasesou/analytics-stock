# /backtest（バックテスト）

## この画面の目的

- 戦略を本番に出す前に、過去データで挙動確認する画面です。

## できること

- 戦略別の成績比較
- DD / Sharpe / CAGR などの確認
- 採用前の安全確認

## 指標の意味

- `Sharpe`
  - 変動（リスク）に対して効率よく利益を出せたか。
- `Max DD`
  - 期間中にどれだけ大きく下がったか。
- `CAGR`
  - 年率換算した成長率。
- `Trade Count`
  - 取引回数。少なすぎると偶然の可能性が高いです。

## 判断の目安

- Sharpeが高く、Max DDが小さい戦略を優先
- 取引回数が極端に少ない戦略は保留

## 注意点

- バックテストは将来を保証しません。
- 必ず `paper` 段階で実運用相当の確認を行ってください。
